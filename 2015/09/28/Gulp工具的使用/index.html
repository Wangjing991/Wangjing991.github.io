Gulp工具的使用

  • gulp

gulp简介

gulp是前端开发过程中对代码进行构建的工具，是自动化项目的构建利器，不仅能对网站资源进行优化，而且在开发过程中很多重复的任务能够使用正确的工具自动完成，使用它，我们不仅可以很愉快的编写代码，同时还可以提高工作效率。

gulp是基于Nodejs的自动任务运行器， 她能自动化地完成javascript/coffee/sass/less/html/image/css 等文件的的测试、检查、合并、压缩、格式化、浏览器自动刷新、部署文件生成，并监听文件在改动后重复指定的这些步骤。在实现上，她借鉴了Unix操作系统的管道（pipe）思想，前一级的输出，直接变成后一级的输入，使得在操作上非常简单。通过本文，我们将学习如何使用Gulp来改变开发流程，从而使开发更加快速高效。

gulp运用

安装gulp

说明：由于gulp是基于nodeJs运行的，所以我们应该在已安装nodeJs的环境操作；nodeJs详见上篇博客：nodeJs初认识

1、使用安装命令npm代码如下：

npm install -g gulp

npm install--save-dev gulp

2.安装完成后，使用gulp -v检查安装是否正确

gulp -v
[16:47:10] CLI version 3.9.1
[16:47:10] Local version 3.9.1

3.运行一个简单的gulp脚本
新建一个gulpfile.js 文件

var gulp = require('gulp')
   gulp.task('default', function() { 
      console.log('this is the default task')
   })

4.使用命令gulp运行

  [16:48:20] Using gulpfile ~/Desktop/gulp/gulpfile.js
[16:48:20] Starting 'default'...
 this is the default task
[16:48:20] Finished 'default' after 110 μs

至此我们已经初步的完成了对gulp的基本了解

深入浅出gulp

1.新建package.json文件
packeage.json是基于nodeJs项目必不可少的配置文件，它是存放于项目根目录的普通json文件
执行命令：npm init初始化package.json文件

运行结果：

{
"name": "gulp",
"version": "1.0.0",
"description": "",
"main": "index.js",
"dependencies": {
"gulp": "^3.9.1"
},
"devDependencies": {},
"scripts": {
"test": "echo \"Error: no test specified\" && exit 1"
},
"author": "",
"license": "ISC"
}

2.本地安装gulp插件
在当前目录下执行以下命令 ：npm install gulp –save-dev本地安装gulp
安装gulp-jsmin插件为例，执行npm install gulp-jsmin –save-dev；
安装完成后将会在node_modules目录下产生gulp-jsmin文件夹

注：全局安装了gulp，项目也安装了gulp，全局安装gulp是为了执行gulp任务，本地安装gulp则是为了调用gulp插件的功能

3.新建gulpfile.js文件

gulpfile.js是gulp项目的配置文件，是位于项目根目录的普通js文件（其实将gulpfile.js放入其他文件夹下亦可）

在gulpfile.js文件写入一些需求：

//导入工具包 require('node_modules里对应模块')
 var gulp = require('gulp'), //本地安装gulp所用到的地方
 var jsmin = require('gulp-jsmin');
 var concat = require('gulp-concat')

//定义一个testJsmin任务（自定义任务名称）
gulp.task('testJsmin', function () {
gulp.src("js/*.js")//该任务针对的文件
    .pipe(concat('all.js'))//合并到新的文件
    .pipe(jsmin()) //该任务调用的模块
    .pipe(gulp.dest('js')); //将会在js下生成all.js
});

gulp.task('default',['testJsmin', 'elseTask']); //定义默认任务

   //gulp.task(name[, deps], fn) 定义任务  name：任务名称 deps：依赖任务名称 fn：回调函数
   //gulp.src(globs[, options]) 执行任务处理的文件  globs：处理的文件路径(字符串或者字符串数组) 
   //gulp.dest(path[, options]) 处理完后文件生成路径

4.在命令行执行：gulp完成对js文件的压缩
